## 약관 핵심 6가지 + 우리 결정

| # | 항목 | 약관 판단 | 우리 결정 |
|---|---|---|---|
| 1 | **결과 캐싱 (5분 in-memory)** | 금지에 가까움 (약관 7.3, 8조 — API로 취득한 정보 무단 복제·저장 금지) | **❌ 네이버 결과는 캐싱 안 함**. 매번 신선한 호출 |
| 2 | **결과 가공 (광고 필터·마크다운 재포맷)** | 조건부 허용 (특약 2.1) — 단순 UI 정리 OK, URL·내용 임의 수정·왜곡 금지 | **✅ 광고 필터 + 마크다운 재포맷 OK**, URL/내용은 원문 그대로 유지 |
| 3 | **무료 공모전 출품작 분류** | 조건부 허용 (FAQ 비용·제휴, 약관 7.2) — 사용자에게 API 이용 대가 안 받으면 무료 이용 범주 | **✅ 무료 공모전 출품작이므로 OK** |
| 4 | **일일 25,000회 초과** | 금지 (자동 차단) — HTTP 429 호출 한도 초과 오류, 제휴 신청 대상 | **✅ 25,000회 한도 내 운영**. 도구함 사용량 모니터링 |
| 5 | **HTTPS 호출 + 헤더 인증** | 조건부 허용 (약관 7.3·7.4) — HTTPS, 등록 환경, 헤더 인증, 자동화 금지, 개인정보 최소화 | **✅ 모두 준수** (`httpx` 기본 HTTPS, 헤더 인증, 개인정보 미전송) |
| 6 | **출처 표기 의무** | 허용 조건 (FAQ API 가이드) — 네이버 오픈 API로 개발된 서비스임을 링크/로고 등으로 표시 | **✅ Tool 결과 마크다운에 "🔍 네이버 검색 기반" 출처 표기 추가** |

## 캐싱 정책 변경 — 아키텍처 영향

**원안** (architecture-overview.md 초안):
- `shared/http_client.py`에 5분 LRU 캐싱
- 같은 query에 대해 5분간 캐시 응답

**수정안**:
- **네이버 API 호출 결과: 캐싱 ❌** (약관 위반 위험 회피)
- **Tavily API 호출 결과: 단기 캐싱 OK** (Tavily 약관 별도, 일반적으로 검색 API 캐싱 허용)
- 동일 query에 대한 동시 호출 보호용 단기 (30초 미만) in-flight deduplication만 적용

### 성능 영향

- 네이버 매번 호출 = 1~2초 추가
- p99 3,000ms 목표는 여전히 만족 가능
- 다만 평균 100ms는 외부 호출 있는 Tool엔 어려움 (이건 외부 호출 없는 Tool로 평균 끌어내림)

## 보안/네트워크 추가 점검

| 항목 | 우리 처리 |
|---|---|
| HTTPS 강제 | ✅ httpx 기본 |
| Client ID/Secret 환경 변수 | ✅ `.env`, .gitignore |
| 자동화 과부하 | ✅ `asyncio.Semaphore`로 동시 5개 제한 |
| 개인정보 전송 | ✅ 검색 query만 (받는 사람 정보는 받은 그대로 사용 안 함) |
| 등록 서비스 환경 일치 | ⚠️ Phase 2에서 PlayMCP 등록 후 카카오클라우드 도메인 일치 확인 |
